## TL;DR

- Claude Code に **「日本語のキャッチコピー・タイトル・X投稿の冒頭フックを15型から生成する」** skill を作って OSS で公開した
- 15型は **5クラスター × 3パターン** に整理。狙い別に「どの型を使うか」を即決できる
- 数値の捏造防止・ニセ締切の禁止など **ガードレール** 内蔵
- リポ: <https://github.com/okazaky/copywriting-15-patterns-skill>

---

## 動機：見出しが弱い理由は「型を1つしか知らないから」

LP・X投稿・noteの見出しを書くたびに、つい同じ型に寄ってしまう感覚はないでしょうか。

私の場合、気を抜くと:

- 数字だけで殴る（`10倍速くなる方法`）
- 問いかけだけで誘う（`まだ◯◯で消耗していますか？`）

の2型に偏っていました。原因はシンプルで、**手元の引き出しが少ない**からです。
「他にどう書けるか」を毎回ゼロから考えるのは認知負荷が重く、結果として一番慣れた型に逃げる。

そこで、私が普段の業務で使い分けている15の型を **Claude Code の Skill 形式** で一つにまとめ、毎回そこから3〜5型を選んで案出ししてもらう構成にしました。

## 出力サンプル：テーマ「個人事業主向けAI業務自動化スクール」

何ができるかが一番伝わりやすいので、先に出力例から。

このskillに「LPの見出しを5クラスターから10案」と頼むと、こう返してきます。

> **01 禁忌ゲート**（Curiosity Gate）
> - 忙しさを売り文句にしたい人は、ここから先は読まないでください。
> - 自分の手を動かす時間が好きな人は、参加しないほうが幸せです。
>
> **02 数値アンカー**（Specificity）
> - ひとり社長が週{N}時間を取り戻す、AI業務自動化の最小設計図。
> - 請求書・見積書・問い合わせ返信。3つだけ自動化して月{N}時間。
>
> **04 反転トリガー**（Curiosity Gate）
> - 「AIで全業務自動化」は無理。本当にやるべきは、3工程だけ手放すこと。
> - 効率化ツールを増やすほど、ひとり社長は遅くなる。
>
> **13 等身大宣言**（Emotional Hook）
> - ITに弱い40代社長が、半年で社内DXを終わらせた手順を公開します。
> - ChatGPTに毎回同じ質問をしていた私が、ようやく辿り着いた最小ワークフロー。
>
> **15 二人称コール**（Reader Anchoring）
> - 来年の今日も、同じ手作業を繰り返すあなたへ。
> - 「時間がない」と書きながら、これを読んでいるあなたへ一言。

「数値アンカー」のところが `{N}` のままなのに気づくと思います。
これは仕様で、**ユーザーが実数を渡していない場合は数字を捏造しない**というガードレールが効いた結果です。

## 設計：5クラスター × 3パターン

15型をそのまま並べると選びにくいので、機能で5つの束にしてあります。

| クラスター | 役割 | 含まれる型 |
|---|---|---|
| **A. Curiosity Gates** | スクロールを止める | 01 禁忌ゲート / 04 反転トリガー / 10 内側開示 |
| **B. Specificity** | 信用度を上げる | 02 数値アンカー / 09 対比軸 / 14 検証ナラティブ |
| **C. Emotional Hooks** | 体温を一段上げる | 05 感情ドライブ / 12 音感ワード / 13 等身大宣言 |
| **D. Reader Anchoring** | 自分事化させる | 03 自己投影クエスチョン / 07 宛名ロック / 15 二人称コール |
| **E. Narrative Push** | 動きを作る | 06 物語の入口 / 08 落とし穴開示 / 11 時限プレッシャー |

実装のレベルでは、Skillに **「3〜5型 / 2クラスター以上から」** を必ず選ぶよう書いてあります。
1クラスターに寄ると案出しが似通うので、**異なるクラスターから2つ以上引く** ことで自動的に多様性が確保される設計です。

## 中身：パターン1つ分のサンプル

すべての型は同じスキーマで書いてあります。例として `pattern-01-prohibition.md` を抜粋:

```markdown
# Pattern 01: 禁忌ゲート (Forbidden Gate)

Cluster: Curiosity Gates

## Effect
「読むな」「やるな」と門を閉じることで、
逆に通り抜けたい衝動を生む。カリギュラ効果を入口設計に使う。

## Best For
- 入口で読者を選別したい時
- ニッチ層への鋭いフック
- 高単価・限定商材の見出し

## Prompt Template
「{{theme}}」について、読者に「やるな・読むな」と
門を閉じる形のコピーを10案出す。

## Shape
- この記事、◯◯な人は閉じてください
- 下手に試さないでください
- 知ってしまうと戻れないので注意

## Quick Example
- 軽い気持ちで始める人は、ここから先は読まないほうがいい。
```

`Effect`（なぜ効くか）→`Best For`（向いている用途）→`Prompt Template`（生成プロンプト）→`Shape`（型のテンプレ表現）→`Quick Example`（完成例）の順。
LLMにとっても人間にとっても「使い方が一目」になるよう統一しています。

## ガードレール

コピーは盛りやすい領域なので、**スキル側で禁じ手を最初から書いて** あります。

- **数字捏造の禁止**: `pattern-02-numbers.md` / `pattern-14-data.md` 内の Guardrail で「ユーザーが提示した実数値、または明らかに構造的な数字（3つの理由・5ステップ等）以外は使わない」
- **ニセ締切の禁止**: `pattern-11-urgency.md` で「実在しない締切や残席数を書かない。期日が無いテーマでは『相対的な遅れ』だけを語る」
- **誇張・煽動の抑制**: SKILL.md 全体の `Constraints` セクションで「Manipulative or deceptive claims を避ける」を明示

これにより、出力が「使うとアカウントBANされそうな広告コピー」にならず、そのまま LP やX に貼れる粒度に揃います。

## 使い方

### Claude Code に手動で入れる場合

```bash
git clone https://github.com/okazaky/copywriting-15-patterns-skill.git \
  ~/.claude/skills/copywriting-15-patterns
```

セッションを再起動すると、以下のような依頼で自動的に発火します。

- 「『◯◯』のLPの見出しを10案ください」
- 「X投稿の冒頭フックを反転トリガーで5案」
- 「Cluster A の3型からそれぞれ3案」

### プラグインとして配布（予定）

`.claude-plugin/plugin.json` を含めてあるので、Claude Code のプラグインマーケットへの登録準備は出来ています。

## まとめ

「型を1つしか持っていない」状態から「型を15個持っていて、5束に整理されている」状態に変わると、見出し作成の体感は **時短よりも安定感の向上** で効きます。
弱い案を出してしまったら別クラスターに振ればいいだけ、という前提があると、初稿の心理的なハードルが一気に下がります。
